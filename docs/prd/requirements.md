# 2. 需求列表 (Requirements)

功能性需求 (Functional):

FR1 (MVP): 用户必须能够创建画布，并选择预设的尺寸比例 (3:4, 1:1, 4:3, 21:9)。

FR2 (MVP): 用户可以在画布上添加文本框，并修改文本内容、字体、字号、颜色和背景色。

FR3 (MVP): 用户可以从本地上传图片，并将其添加到画布中。

FR4 (MVP): 画布上的所有元素（文本、图片、形状）都支持拖拽移动、缩放和旋转。

FR5 (MVP): 用户可以设置画布的背景（纯色）。

FR6 (MVP): 用户可以通过快捷键绘制基本形状（R-矩形, O-圆形, L-直线）。

FR7 (MVP): 用户必须能够将当前画布内容"复制"到系统剪贴板。

FR8 (MVP): 用户必须能够将当前画布内容"下载"为 PNG 格式的图片文件。

FR9 (P2): 用户应能从一个包含至少18种精选免费中文字体的列表中选择字体。

FR10 (P2): 用户应能设置画布背景为渐变色（包括预设和马卡龙色）。

FR11 (P2): 用户应能从 Lucide 图标库中选择并添加图标到画布。

FR12 (P2): 用户应能对选中的图片应用高级编辑：去背景、圆角、阴影、滤镜、裁剪。

FR13 (P3): 用户应能通过输入提示词，使用 AI (Seedream 4.0) 生成图片并添加到画布。

FR14 (P3): 用户应能选中画布中的图片，通过输入提示词对其进行风格修改。

FR15 (P3): 用户应能从 Unsplash 图库搜索并添加图片到画布。

FR16 (P4): 用户应能将自己的设计稿保存到 Vercel 数据库。

FR17 (P4): 用户应能生成一个唯一的URL来公开分享他们的设计稿。

FR18 (P4): 用户应能选择将自己的设计稿"共享为模板"。

FR19 (P4): 用户应能选择将自己的图片"共享为素材"。

非功能性需求 (Non-Functional):

NFR1 (MVP): 页面加载性能应优化，首次加载时间（LCP）应在 2.5 秒以内。

NFR2 (MVP): 应用应在主流浏览器（Chrome, Safari, Firefox）的最新版本上表现一致。

NFR3 (MVP): 所有第三方字体和素材库必须使用CDN链接。

NFR4 (P2+): 所有第三方API（AI、图库等）调用必须有明确的错误处理和超时机制。

NFR5 (P2+): 用户的API密钥（如 Seedream）必须安全存储在客户端（如 localStorage），不得传输到服务器。
