# 回归测试用例

## 回归测试范围

在每次发布新功能后，需要执行以下回归测试确保现有功能不受影响。

## 核心功能回归测试

### 1. 画布基础功能
- [ ] 画布创建和尺寸调整
- [ ] 文字添加和编辑
- [ ] 图片上传和操作
- [ ] 形状绘制（矩形、圆形、直线）
- [ ] 元素拖拽、缩放、旋转

### 2. 导出功能
- [ ] 复制到剪贴板
- [ ] 下载为 PNG
- [ ] 导出内容完整性验证

### 3. 界面交互
- [ ] 工具栏所有按钮可点击
- [ ] 左侧面板切换正常
- [ ] 右侧属性面板动态更新
- [ ] 快捷键响应正常（R/O/L）

### 4. 性能回归
- [ ] 页面加载时间 < 2.5 秒
- [ ] 操作响应时间 < 500ms
- [ ] 内存使用稳定

## 兼容性测试

### 浏览器兼容性
- [ ] Chrome 最新版本
- [ ] Safari 最新版本
- [ ] Firefox 最新版本
- [ ] Edge 最新版本

### 设备兼容性
- [ ] 桌面端（1920x1080）
- [ ] 桌面端（1366x768）
- [ ] 平板端（iPad）
- [ ] 移动端（iPhone/Android）

## 安全性测试

### 数据安全
- [ ] 用户数据加密存储
- [ ] API 密钥安全存储（localStorage）
- [ ] 公开分享链接安全性

### 权限控制
- [ ] 模板共享权限
- [ ] 素材共享权限
- [ ] 数据访问控制

## 用户体验回归

### 易用性
- [ ] 零学习成本验证
- [ ] 操作流程直观性
- [ ] 错误提示清晰性

### 反馈机制
- [ ] 操作成功提示
- [ ] 错误状态提示
- [ ] 加载状态提示

## 自动化回归测试

### 持续集成检查
- [ ] 单元测试通过率 100%
- [ ] 集成测试全部通过
- [ ] 端到端测试关键路径

### 回归测试报告
每次回归测试完成后，记录：
- 测试执行日期
- 测试执行人
- 通过的测试用例数量
- 失败的测试用例数量
- 发现的问题列表
- 问题修复状态

## 回归测试检查清单

发布前必须确认：
- [ ] 所有 MVP 功能正常工作
- [ ] 最新功能与现有功能兼容
- [ ] 无性能回归
- [ ] 无安全漏洞
- [ ] 跨浏览器兼容性正常
- [ ] 移动端体验良好
- [ ] 回归测试报告已生成

## 问题跟踪

所有在回归测试中发现的问题需要：
1. 在问题跟踪系统中创建问题单
2. 分配给相应的开发人员
3. 跟踪修复进度
4. 验证修复结果
5. 更新回归测试状态
